# MCP-сервер с планировщиком — напоминания и фоновые уведомления

## Что сделано

Мы написали MCP-сервер-планировщик и подключили его к CLI. Сервер позволяет агенту управлять напоминаниями: создавать одноразовые и повторяющиеся, просматривать просроченные и отмечать выполненные — всё через естественный язык.

Главная особенность: **уведомления приходят сами**, без лишних вопросов. Пока пользователь просто набирает текст или ждёт — сервер в фоне следит за временем и показывает уведомление прямо над строкой ввода, не прерывая диалог.

---

## Как это работает

```
                    ┌──────────────────────────────────┐
                    │   CLI (cli-app)                   │
                    │                                   │
   пользователь ───▶│  Основной цикл ввода              │
                    │                                   │
     ● уведомление  │  Фоновый читатель уведомлений     │
         ◀──────────│  → показывает над строкой ввода   │
                    └──────────────┬───────────────────┘
                                   │ JSON-RPC команды
                                   │ [NOTIFY] уведомления
                    ┌──────────────▼───────────────────┐
                    │   mcp-server-scheduler            │
                    │                                   │
                    │  ← команды LLM (создать/показать) │
                    │  Фоновый тик (каждые N секунд):   │
                    │    автоперенос повторяющихся  ─────▶ [NOTIFY]
                    │    проверка срока одноразовых ─────▶ [NOTIFY]
                    │                                   │
                    │  Хранит данные в reminders.json   │
                    └───────────────────────────────────┘
```

**Сервер** работает в двух режимах одновременно:
- Отвечает на команды агента: создать напоминание, показать список, отметить выполненным
- В фоне, независимо от диалога, каждые N секунд проверяет — не наступил ли срок. Когда срок приходит, сервер отправляет уведомление, и оно появляется на экране

**CLI** получает уведомления через отдельный канал, который не мешает обычному JSON-потоку с командами. Уведомление отображается **над строкой ввода** — пользователь видит его, даже если сейчас что-то набирает.

Данные сохраняются в `~/.llmchat/reminders.json` и переживают перезапуск CLI.

## Инструменты сервера

| Инструмент          | Параметры                                                                                                                            | Описание                                                                                  |
|---------------------|--------------------------------------------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------|
| `schedule_reminder` | `title`, `scheduled_at` (ISO-8601, или по-русски: «15 марта 2026 09:00»), `description?`, `recurrence?`, `recurrence_interval_days?` | Создаёт напоминание — одноразовое или повторяющееся                                       |
| `list_reminders`    | `include_dismissed?`                                                                                                                 | Показывает все активные напоминания (и выполненные, если нужно)                           |
| `get_due_reminders` | —                                                                                                                                    | Возвращает просроченные одноразовые напоминания — те, что ждут подтверждения пользователя |
| `cancel_reminder`   | `id`                                                                                                                                 | Удаляет напоминание навсегда                                                              |
| `dismiss_reminder`  | `id`                                                                                                                                 | Отмечает выполненным; для повторяющихся — переносит на следующую дату                     |
| `get_trigger_log`   | —                                                                                                                                    | Журнал: что фоновый планировщик сделал автоматически за текущую сессию                    |

Поддерживаемые режимы повторения (`recurrence`): `none`, `daily`, `weekly`, `monthly`.

### Разница между одноразовыми и повторяющимися

| Тип           | Что происходит, когда приходит срок                                        |
|---------------|----------------------------------------------------------------------------|
| Одноразовое   | Остаётся как «просроченное» до явного подтверждения пользователем          |
| Повторяющееся | Сервер **автоматически** переносит на следующую дату и записывает в журнал |

## Руководство по использованию

### 1. Сборка сервера

```bash
./gradlew :mcp-server-scheduler:shadowJar
```

### 2. Подключение из CLI

```
/mcp connect java -jar mcp-server-scheduler/build/libs/mcp-server-scheduler-1.0-SNAPSHOT-all.jar
```

CLI запускает сервер, выполняет рукопожатие и передаёт список инструментов агенту. С этого момента сервер работает в фоне и следит за временем.

### 3. Просмотр инструментов

```
/mcp tools
```

### 4. Работа с агентом

После подключения — общайтесь на естественном языке, в том числе с датами по-русски:

```
Запланируй напоминание на 15 марта 2026 10:00 — "Позвонить в банк"
Создай ежедневное напоминание на 16 марта 2026 09:00 — "Утренняя зарядка"
Покажи все активные напоминания
Есть ли просроченные задачи?
Отметь напоминание про банк как выполненное
Что планировщик сделал автоматически?
```

### 5. Уведомления

Уведомления появляются **сами**, без запроса. Когда срок напоминания наступает, над строкой ввода появляется:

```
────────────────────────────────────────────────────────────
 ● Позвонить в банк
────────────────────────────────────────────────────────────
```

### 6. Отключение

```
/mcp disconnect
```

## Плюсы и минусы

### Плюсы

**Уведомления не прерывают работу.** Они появляются над строкой ввода — пользователь замечает их в удобный момент, а не в виде всплывающего окна, сбивающего фокус.

**Повторяющиеся задачи без накопления ошибок.** Перенос дат использует календарную арифметику, а не сложение секунд — месячные напоминания не смещаются со временем.

**Журнал автоматических действий.** `get_trigger_log` показывает, что сервер сделал сам: когда что перенёс, на какую дату. Пользователь всегда может спросить: «что ты сделал, пока я отсутствовал?»

**Персистентность без базы данных.** Данные хранятся в `~/.llmchat/reminders.json` — тот же подход, что у памяти и инвариантов. Никакой инфраструктуры, никаких миграций.

### Минусы

**Уведомления только пока сессия открыта.** Если CLI закрыт, уведомления никуда не доставляются. Для по-настоящему стойких напоминаний нужен системный демон или внешний сервис.

**Нет временны́х зон.** Дата интерпретируется в локальном времени машины. При работе с удалёнными серверами или разными часовыми поясами это может привести к путанице.

**Один экземпляр.** Запуск двух копий сервера одновременно приведёт к конфликту при записи в файл. На практике это не проблема: CLI держит только одно MCP-подключение.

**31-е число в коротких месяцах.** Ежемесячное напоминание на 31-е число в феврале сдвинется на 28-е или 29-е — стандартное поведение календарной арифметики.

## Архитектура модуля

```
mcp-server-scheduler/
├── Main.kt              # точка входа
├── McpServer.kt         # JSON-RPC цикл + фоновый планировщик + 6 инструментов
├── Reminder.kt          # модели данных: Reminder, Recurrence, TriggerEvent
└── ReminderStorage.kt   # хранение и автоперенос повторяющихся напоминаний
```

### Переменные окружения

| Переменная              | По умолчанию | Описание                                   |
|-------------------------|--------------|--------------------------------------------|
| `SCHEDULER_INTERVAL_MS` | 60 000       | Как часто (в мс) сервер проверяет дедлайны |
